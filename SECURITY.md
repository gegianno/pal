# Security Policy

## Reporting a vulnerability

Please do **not** open a public issue for security-sensitive reports.

Preferred: use GitHub Security Advisories (private vulnerability reporting) for this repository.

If that is not available for any reason, contact the maintainer via their GitHub profile with:

- a clear description of the impact
- a minimal reproduction (if applicable)
- affected versions (if known)

## Supported versions

Security fixes are provided for the latest released version.
